Causal Mosaic Schema

A practical guide to CAMO 0.7.3: how ecological causal claims become validated LinkML records that can power evidence maps, causal diagrams, fuzzy cognitive maps, retrieval systems, and practitioner summaries.

Version 0.7.3 Source: camo-0.7.3.yaml Example: Murphy 2005
Start Here

What CAMO Is

The Causal Mosaic Schema is a LinkML data model for recording causal evidence in restoration ecology. It is not an app, not a general-purpose ontology, and not a pile of prose notes. It is a versioned contract for turning claims from literature into a graph that people and software can inspect consistently.

Schema

CAMO defines the classes, slots, ranges, and controlled vocabularies used by annotations. The hand-authored YAML is the source of truth.

Graph

Each annotation becomes a labeled property graph: nodes are ecological states or changes, edges are causal claims between those states or changes.

Evidence

Each edge records how strong the claim is, which causal accounts it invokes, which features are present, and what kind of evidence supports it.

The Core Idea

Causation Is Between Changes, Not Bare Things

CAMO's most important modeling rule is simple: in ecological evidence, a species, process, or place does not usually cause something by itself. The causally relevant unit is a state or change in a measurable attribute.

Entity What thing is involved? entity_term
→
Attribute What property is being measured? measured_attribute
→
Qualifier How is that property stated or changed? state_or_change_qualifier

A node such as "decreased reproductive output of Alliaria petiolata" decomposes into a taxon, a measured attribute, and a change qualifier. This lets a computer find all claims involving garlic mustard, all claims about reproductive output, or all claims involving decreases, without losing the human-readable label.

Cause node Introduction of transplanted Sanguinaria canadensis entity_type: management_intervention
state_or_change_qualifier: introduced
causal edge negatively_regulates
Effect node Decreased population performance of Alliaria petiolata entity_type: taxon
measured_attribute: population performance
Structure

The Schema Map

A CAMO record is a CausalGraph. It contains provenance, nodes, article-level edges, and optionally synthesis-level evidence assessments.

Class What It Represents When You Use It
CausalGraph The complete annotation graph and its metadata. Every annotation file starts here.
GraphProvenance Project, protocol, version, creation, and corpus metadata. To document where the annotation came from and which framework version it uses.
CausalNode An ecological state or change: entity plus attribute plus qualifier. For causes, effects, mediators, and moderators.
CausalEdge An article-level causal claim connecting a subject node to an object node. For each causal relationship extracted from a paper or report.
EvidenceBaseAssessmentEdge A synthesis-level assessment across multiple comparable article-level edges. For systematic reviews, evidence maps, and aggregate causal judgments.

Top-Level YAML Shape

graph_id: camo_example_graph
schema_version: 0.7.3
provenance:
  ontology_framework: ELMO + CAMO
  causal_mosaic_version: 0.7.3
  project: EcoWeaver
nodes:
  - id: node:example_cause
    name: "Increased abundance of example taxon"
    entity_type: taxon
    entity_term: Q12345
    measured_attribute: abundance
    state_or_change_qualifier: increased
  - id: node:example_effect
    name: "Decreased cover of example invasive plant"
    entity_type: taxon
    entity_term: Q67890
    measured_attribute: cover
    state_or_change_qualifier: decreased
edges:
  - id: edge:example_claim
    subject: node:example_cause
    predicate: causes
    object: node:example_effect
    claim_strength: direct_causal
    philosophical_accounts:
      - interventionist
    source_spans:
      - text: "Short source span grounding this claim."
Nodes

How to Read a CausalNode

A CausalNode is the reusable unit of ecological meaning. It should say enough to be understandable as a label, but it should also be decomposed into structured fields so software can compare claims across papers.

Slot Range Plain-Language Meaning
id uriorcurie Stable node identifier, usually generated by the annotation tool.
name string Human-readable label that expresses the full node meaning.
entity_type EntityTypeEnum Broad kind of entity: variable, process, intervention, or taxon.
entity_term uriorcurie or string The thing being described: a taxon, process, chemical, habitat, or variable.
measured_attribute uriorcurie or string The measured property, such as abundance, cover, height, density, or reproduction.
state_or_change_qualifier StateOrChangeQualifierEnum The state or direction: increased, decreased, present, introduced, removed, and related values.
taxonomic_scope uriorcurie or string, multivalued Scopes an environmental variable to a taxon set or guild, such as native understory species.
source_spans TextSpan, multivalued Grounding snippets or local span references that explain why the node exists.
annotator string Human, ORCID, model, or workflow responsible for the node annotation.

Entity Types

environmental_variable

A measurable aspect of ecosystem state: soil pH, resource availability, canopy cover, species richness, water table level.

environmental_process

A process that changes ecosystem state over time: deposition, decomposition, competition, succession, herbivory.

management_intervention

A deliberate human action: transplanting, prescribed burning, invasive species removal, seeding, hydrological restoration.

taxon

A single taxon whose attribute is being measured. Use the taxon as entity_term and the property as measured_attribute.

Taxon-Scope Two-Path Model

CAMO 0.7.3 adds taxonomic_scope so multi-taxon measurements can be represented without adding non-causal graph edges.

Situation Use Example
Single-taxon measurement entity_type: taxon entity_term: Alliaria petiolata and measured_attribute: reproductive output
Multi-taxon aggregate or ecological guild entity_type: environmental_variable plus taxonomic_scope entity_term: re-establishment with taxonomic_scope: [native forest understory species]
Intervention or process directed at an entity Use the active schema's target-scoping pattern when available Transplanting as an intervention directed at Sanguinaria canadensis

State and Change Qualifiers

  • increased
  • decreased
  • present
  • absent
  • introduced
  • removed
  • unchanged
  • unspecified
  • occurred
  • initiated
  • terminated
  • ongoing
Edges

How to Read a CausalEdge

A CausalEdge is the main annotation unit. It connects one node to another and then carries four layers of interpretation: language strength, philosophical account, causal features, and evidence.

Layer Slots Question It Answers
Graph triple subject, predicate, object, negated What causal relationship is being claimed, and is the relation denied?
Layer 1 claim_strength, original_sentence, ecosystem_context How strongly does the source text phrase the causal claim, and where was it observed?
Layer 2 philosophical_accounts Which accounts of causation does the claim rely on?
Layer 3 Fifteen causal feature slots What is said about necessity, direction, mechanism, context, reversibility, and related features?
Layer 4 evidential_basis What kind of evidence supports the claim, and what is the evidence evidence of?
Rendering and provenance fcm_weight, temporal_extent, source_document, annotation_confidence How should downstream systems model, filter, and trace the claim?

Predicate Vocabulary

Use controlled predicates. They are grounded in CAMO, Biolink, or RO where possible and carry signs for downstream fuzzy cognitive maps.

Predicate Sign Use When
causes positive C directly produces or increases E.
contributes_to positive C is one contributing factor among others.
prevents negative C blocks, inhibits, or prevents E.
enables positive C facilitates E but is not necessarily sufficient.
disrupts negative C degrades, suppresses, or disturbs E.
regulates variable C modulates E but the sign is context-dependent or not fixed.
positively_regulates positive C upregulates, increases, or strengthens E.
negatively_regulates negative C downregulates, decreases, or suppresses E.
associated_with / correlated_with unspecified The source reports a relationship without causal commitment.
mediates / moderates pathway / modifier A node lies on a pathway or modifies another causal relationship.

Claim Strength

no_relationship

The source explicitly reports no effect or no relationship.

associational

The source reports association, correlation, or co-occurrence without causal language.

uncertain_causal

The source uses hedged causal language or a conditional causal interpretation.

direct_causal

The source uses unhedged causal language that asserts causation.

Layer 3

The Fifteen Causal Features

CAMO keeps causal interpretation explicit. Most feature slots use a four-value status: explicitly_asserted, implicitly_assumed, explicitly_denied, or not_addressed. More specialized features use their own enums or small inlined structures.

  • necessity Is C required for E?
  • sufficiency Is C enough to produce E?
  • direction Is C-to-E direction asserted, uncertain, or bidirectional?
  • temporal_ordering Does the source establish that C precedes E?
  • mediation Is a pathway or mediator identified?
  • moderation Does a third variable alter the effect?
  • strength Is magnitude, effect size, or dose response reported?
  • specificity Is the relation specific to a cause, effect, or context?
  • stability Is the relationship stable across settings or times?
  • token_or_type Is the claim about one case or a general relation?
  • determinism Is causation framed as deterministic, probabilistic, or ambiguous?
  • proximate_distal Is the cause immediate, upstream, or both?
  • contributing_sole Is C one factor or the only cause?
  • reversibility Would removing C reverse E?
  • proportionality Is the cause described at the right level of specificity?

Inlined Feature Structures

Structure Useful Fields Example Use
DirectionAnnotation status, evidence_for_direction Direction established by a randomized field experiment.
MediationAnnotation status, mediator_node_ids, pathway_description Competition is inferred through reduced resource availability.
ModerationAnnotation status, moderator_node_ids, interaction_type Effect magnitude changes with treatment density.
StrengthAnnotation quantitative_value, qualitative_descriptor, dose_response A strong, dose-responsive suppressive effect is reported.
ContextAnnotation scope_conditions, geographic_scope, temporal_scope, ecosystem_scope The claim is scoped to a degraded deciduous forest edge in Ontario during 1997-2000.
Layer 4

Evidence Basis

CAMO separates the type of evidence from the object of evidence. This distinction makes it possible to see whether a paper supports difference-making, production/mechanism, or both.

evidence_types

What kind of study or reasoning produced the evidence? Examples include randomized_experiment, observational_longitudinal, mechanistic_study, structural_equation_model, meta_analysis, modeling_simulation, expert_judgment, and practitioner_experience.

evidence_objects

What is the evidence evidence of? Use correlation for difference-making evidence, mechanism for production evidence, and both when the source supports both sides.

Philosophical Accounts

CAMO's causal mosaic layer lets one edge invoke more than one account of causation. A management experiment, for example, may be interventionist because a treatment was manipulated, probabilistic because outcomes vary across plots, and agency because the claim matters for management action.

Account Family Question
counterfactual difference-making Would E have occurred without C?
probabilistic difference-making Does C change the probability or systematic variation of E?
interventionist difference-making Would intervening on C change E?
transmission production What material, signal, energy, or information passes from C to E?
mechanistic production What entities and activities produce E from C?
regularity complementary Does C-type regularly precede E-type?
inus_component complementary Is C a necessary part of one sufficient causal constellation?
agency complementary Can an agent bring about E by doing C?
Worked Example

Murphy 2005: Bloodroot, Garlic Mustard, and Restoration

The supplied example annotation is based on Stephen D. Murphy's 2005 Restoration Ecology article, "Concurrent Management of an Exotic Species and Initial Restoration Efforts in Forests." The study tested whether transplanted bloodroot, Sanguinaria canadensis, could suppress garlic mustard, Alliaria petiolata, in a degraded deciduous woodlot edge near Waterloo, Ontario.

Study Design

Randomized complete blocks with 40 one-square-metre plots and ten bloodroot density treatments observed across four growing seasons.

Main CAMO Claim

Transplanted bloodroot negatively regulates garlic mustard population performance and reproductive output in the studied forest-edge context.

Important Limit

The annotation also records a null or delayed outcome: broader native understory re-establishment was not detected by the end of the study.

From Paper to CAMO

Ecological Idea CAMO Representation Why It Matters
Bloodroot was transplanted at different densities. A management_intervention node with qualifier introduced. The intervention is represented as the cause-side node, not as a loose study note.
Garlic mustard performance decreased. A taxon node for Alliaria petiolata with measured_attribute: population performance and qualifier decreased. The taxon and measured property stay separable for search and synthesis.
Resource competition is the inferred pathway. A mediator node for decreased resource availability, referenced by mediation.mediator_node_ids. The annotation distinguishes observed treatment effects from inferred mechanism.
Density modifies effect magnitude. moderation.status: explicitly_asserted and interaction_type: quantitative. The edge can record that more of the intervention does not always mean a simple linear benefit.
Suppression did not yet produce broad native recovery. A negated edge with claim_strength: no_relationship. Null results are still evidence and should remain visible in synthesis.

Example Edge, Condensed

- id: edge:murphy2005_sanguinaria_suppresses_alliaria_performance
  subject: node:murphy2005_sanguinaria_transplanting
  predicate: negatively_regulates
  object: node:murphy2005_alliaria_population_performance
  claim_strength: direct_causal
  ecosystem_context: deciduous_temperate_forests
  philosophical_accounts:
    - interventionist
    - probabilistic
    - agency
  direction:
    status: asserted
    evidence_for_direction: experimental
  mediation:
    status: implicitly_assumed
    mediator_node_ids:
      - node:murphy2005_resource_availability_to_alliaria
  moderation:
    status: explicitly_asserted
    interaction_type: quantitative
  strength:
    status: explicitly_asserted
    qualitative_descriptor: strong
    dose_response: true
  evidential_basis:
    evidence_types:
      - randomized_experiment
    evidence_objects:
      - both
  temporal_extent:
    duration_months: 48
    observation_grain: seasonal
What the example teaches. Do not make a separate causal node for every treatment density unless the density itself is the causal object of analysis. In this annotation, density is captured as strength, moderation, and context metadata on the edge.
Annotating

A Practical Annotation Workflow

  1. Find causal language and causal design. Look for explicit claims, null effects, inferred mechanisms, experimental contrasts, temporal ordering, or management recommendations.
  2. Create the nodes first. For each cause, effect, mediator, and moderator, write the full human label and then decompose it into entity, measured attribute, and qualifier.
  3. Choose the edge predicate. Pick the controlled predicate that best represents the direction and sign of the claim: causes, negatively_regulates, enables, prevents, and so on.
  4. Set claim strength from the text. This is about the source's wording, not your confidence. Hedged causal language is uncertain_causal; direct causal language is direct_causal.
  5. Assign causal accounts. Ask whether the claim is interventionist, probabilistic, mechanistic, agency-oriented, regularity-based, or some combination.
  6. Fill the causal feature layer conservatively. Mark only what the source states or strongly implies. Use not_addressed when the paper does not speak to a feature.
  7. Record evidence and provenance. Capture the study design, evidence type, evidence object, source document, temporal extent, ecosystem context, and source spans.
  8. Validate against the schema. CAMO data should pass LinkML validation and the repository validation loop.
Checks

Quality Checklist

  • Every node is a state or change, not just a bare species, process, or place.
  • Every node has a useful name, entity_term, and, where applicable, measured_attribute.
  • Single-taxon measurements use entity_type: taxon; multi-taxon or guild measurements use taxonomic_scope.
  • Every edge has subject, predicate, object, claim_strength, and philosophical_accounts.
  • Null results are represented with claim_strength: no_relationship and, when appropriate, negated: true.
  • Feature statuses distinguish explicit claims from assumptions and from silence.
  • Mechanisms are marked as inferred when the source does not directly test them.
  • evidential_basis.evidence_types describes study design; evidence_objects describes what the evidence supports.
  • Source document metadata is present enough for citation, filtering, and later retrieval.
  • The file passes the repo validation workflow before it is released.

Common Pitfalls

Pitfall Better CAMO Habit
Annotating "garlic mustard" as an effect. Annotate "decreased reproductive output of garlic mustard" or the exact measured state/change.
Treating all experimental claims as mechanistic. Experimental evidence often supports difference-making; mechanism needs pathway evidence or a clearly stated mechanistic interpretation.
Using free-text predicates. Choose from CausalPredicateEnum so downstream rendering and FCM signs stay consistent.
Encoding context as extra causal edges. Use ecosystem_context, context_dependence, temporal_extent, and scope fields unless the relation is truly causal.
Letting old field names leak into data. Use current 0.7.3 names, especially measured_attribute rather than older names.
Governance

How to Contribute to CAMO

CAMO is a schema repository. The safest contributions preserve the hand-authored YAML as the source of truth, keep released names stable, and regenerate derived artifacts only from the schema.

Before Changing the Schema

  • Open an issue or proposal for new classes, slots, enums, or breaking changes.
  • Check whether the concept belongs in CAMO, ELMO, another ontology, or project-specific annotation guidance.
  • Remember that class, slot, and enum names are downstream public contracts.

When Editing

  • Edit the LinkML YAML, not generated outputs.
  • Give every class and slot a clear description.
  • Give every slot an explicit range.
  • Declare prefixes and set identifier prefixes where needed.

Release Discipline

  1. Use closed enums for causal categories rather than free-text values.
  2. Do not rename or remove released classes, slots, or enums without a deprecation path.
  3. Record breaking changes in CHANGELOG.md and include migration notes for downstream users.
  4. Run the validation loop before handing off changes: ./validate.sh.
  5. Submit a focused pull request to github.com/timalamenciak/causalmosaic.
Contribution test. A good CAMO contribution makes annotation more precise without making downstream data harder to validate, query, or migrate.
